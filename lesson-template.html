Lesson 01

開発環境構築と実務の基礎

プロジェクトのセットアップから実務フローまで

所要時間: 3-4時間
難易度: 初級
実務レベル: 必須
Docker Git 環境構築 チーム開発

学習目標

  • Docker環境でLaravelプロジェクトをセットアップできる
  • Gitを使った基本的なバージョン管理ができる
  • プルリクエストの流れを理解する
  • 環境変数とセキュリティの基本を学ぶ
  • デバッグツールの使い方をマスターする

実務での重要性

環境構築は開発の第一歩です。特にSES出向先では、プロジェクトごとに異なる環境で開発することが多く、 素早く環境をセットアップできるスキルが求められます。

実務ポイント

占いWebアプリのような既存プロジェクトに参画する場合、まず行うのが環境構築です。 Dockerを使うことで「私の環境では動きます」問題を防ぎ、チーム全員が同じ環境で開発できます。

学習内容

1. Docker環境のセットアップ

Dockerを使ってLaravel開発環境を構築します。

# docker-compose.yml
version: '3.8'

services:
  app:
    image: php:8.2-fpm
    volumes:
      - ./:/var/www/html
    environment:
      - DB_HOST=mysql
      - DB_DATABASE=fortune_app
      
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: fortune_app
      MYSQL_ROOT_PASSWORD: secret
    ports:
      - "3306:3306"

注意点

本番環境のパスワードは絶対にGitにコミットしないでください。 開発環境でも、.envファイルは.gitignoreに追加しましょう。

2. Gitでのバージョン管理

チーム開発に必須のGit操作を学びます。

# ブランチを作成して切り替え
git checkout -b feature/add-fortune-calculation

# 変更をステージング
git add .

# コミット（わかりやすいメッセージで）
git commit -m "feat: 占い結果計算ロジックを追加"

# リモートにプッシュ
git push origin feature/add-fortune-calculation

ベストプラクティス

コミットメッセージは「何を」「なぜ」したのかを明確に。 Conventional Commits（feat:, fix:, docs: など）を使うとチームで統一しやすくなります。

3. 環境変数の管理

.envファイルで機密情報を管理します。

# .env.example（Gitにコミット OK）
APP_NAME=FortuneApp
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=fortune_app
DB_USERNAME=root
DB_PASSWORD=secret  # 本番では変更必須！

演習問題

1

環境構築の実践

以下の手順で開発環境を構築してください：

  1. Dockerをインストール
  2. Laravelプロジェクトをクローン
  3. docker-compose upでコンテナ起動
  4. .env.exampleを.envにコピーして設定
  5. ブラウザでhttp://localhost:8000にアクセス
2

Gitブランチ操作

新機能開発のシミュレーション：

  1. feature/user-profile ブランチを作成
  2. routes/web.phpに新しいルートを追加
  3. 変更をコミット
  4. mainブランチにマージ

確認問題

  1. Dockerを使うメリットを3つ挙げてください
  2. .envファイルをGitにコミットしてはいけない理由は？
  3. git pullとgit fetchの違いを説明してください
  4. コンフリクトが発生した時の対処法は？
解答を表示

1. Dockerのメリット：

  • 環境の統一（全員が同じ環境で開発できる）
  • セットアップが簡単（docker-compose upだけ）
  • 本番環境との差異を減らせる

2. .envファイルの理由：

パスワードやAPIキーなどの機密情報が含まれるため、公開リポジトリに上げると情報漏洩のリスクがあります。
